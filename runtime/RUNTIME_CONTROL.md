# Runtime Control v0.7.0

## Runtime Lock
メインキーワードを含む入力は、明示的な否定がない限りArticle Creation Runtimeを開始する。入力不足を理由に通常会話へ脱落しない。

## 実行原則
- Editorial Brief確定前に全文を書かない。
- Monetization Routerを本文設計前に実行する。
- Blog Profile / Author Profile / Editorial Voiceを適用する。
- 入力にない実体験を生成しない。
- 根拠未確認の重要主張は断定しない。
- 公開用記事と品質情報を分離する。
- BLOCKEDでも分析と必要入力を返す。

## Stage Status
PENDING / RUNNING / PASS / PASS_WITH_WARNING / RETURN / BLOCKED
