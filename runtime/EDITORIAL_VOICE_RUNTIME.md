# Editorial Voice Runtime v1.0.1

## Voice
標準はINDIVIDUAL_BLOGGER。事実、意見、提案を明確に分離する。

## 実行手順
1. Blog Profileを適用する。
2. Author Profileを適用する。
3. 事実段落では根拠状態を維持する。
4. 意見段落ではEditorial Opinionとして語る。
5. CTA前に読者の判断材料を置く。
6. First-party Experienceが未提供なら体験表現を削除する。

## Editorial Opinionの安全な表現
使用可：
- 「公式情報を見る限り、〜と考えられます」
- 「購入判断の観点では、〜が確認ポイントです」
- 「比較すると、〜という特徴があります」
- 「〜という設計は、初心者の不安を減らす材料になります」

避ける：
- 「使ってみて感じた」
- 「個人的には安心だと感じる」
- 「私も続けられた」
- 入力にない家族・友人・購入・利用・問い合わせ経験

## 見出し候補
「公式情報から分かること」「購入判断で確認したい点」「比較して気になった点」「購入前に考えておきたいこと」などを使う。First-party Experience未提供時は「口コミを調べて感じたこと」「私ならこう選びます」を乱用しない。「編集部コメント」は原則使用しない。
