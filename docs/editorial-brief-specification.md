# Editorial Brief Specification v1.0

Editorial Briefは本文生成前に必ず確定する中間成果物である。

## 必須項目

- main_keyword
- supporting_keywords
- primary_intent
- secondary_intents
- reader_profile
- reader_situation
- explicit_problem
- latent_needs
- desired_reader_outcome
- article_type
- article_goal
- central_answer
- mandatory_topics
- optional_topics
- excluded_topics
- original_value_proposition
- evidence_requirements
- freshness_requirements
- tone_and_depth
- monetization_policy
- internal_link_policy
- risk_flags
- missing_information

## 品質条件

1. 中心回答が一文で説明できる。
2. 各必須論点が読者課題に結びつく。
3. キーワードを含めるためだけの論点を置かない。
4. 独自価値は実在する入力・構造・説明方法に基づく。
5. 体験価値を使う場合、提供済みの実体験だけを指定する。
6. 根拠未確認の主張を「確定事実」として計画しない。
7. 収益導線は読者の次の合理的行動と一致させる。

## Brief判定

- BRIEF_READY
- BRIEF_READY_WITH_WARNINGS
- BRIEF_NEEDS_INPUT
- BRIEF_BLOCKED
