# Article Blueprint Design v1.0

Blueprintは固定テンプレートではなく、記事タイプ別の論理順序と必須品質条件を定義する。

## 初期Blueprint

| Blueprint | 中心構造 |
|---|---|
| HOW_TO | 結果 → 前提 → 手順 → 確認 → 失敗対処 |
| TROUBLESHOOTING | 症状 → 切り分け → 原因 → 対処 → 再発防止 |
| EXPLANATION | 定義 → 仕組み → 影響 → 具体例 → 要点 |
| COMPARISON | 比較結論 → 比較条件 → 項目別比較 → 向く人 |
| BUYING_GUIDE | 選び方 → 判断軸 → 候補 → 注意点 → CTA |
| LISTICLE | 選定基準 → 一覧 → 個別解説 → 選び分け |
| LOCAL_GUIDE | 対象地域 → 利用条件 → 選択肢 → 行動情報 |
| NEWS_EXPLANATION | 何が起きたか → 背景 → 影響 → 今後 |
| EXPERIENCE_SUPPORT | 実体験範囲 → 観察 → 一般化しない示唆 |
| YMYL_INFORMATION | 一般情報 → リスク → 公的根拠 → 専門家確認 |

## 複合Blueprint

Primary Blueprintを1つ、Secondary Blueprintを最大2つ選択できる。Secondaryは記事全体を支配せず、必要セクションにのみ適用する。

## Blueprint禁止事項

- 全記事に「おすすめ○選」を適用しない。
- FAQで本文不足を埋めない。
- 比較条件の異なる対象を同列比較しない。
- 体験がないのにEXPERIENCE_SUPPORTを主型にしない。
- YMYLで広告CTAを結論より優先しない。
